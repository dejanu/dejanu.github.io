Important

Fundamental technical inquiries for SRE/DevOps expertise

  • List containers using Pretty-print containers with a Go template instead of using | and awk:
    • docker ps --format "{{.Names}} with {{.Status}}"
  • Remove unused/dangling images and stopped containers? (filter comes in handy). Remove stopped containers and remove unused images not just dangling ones:
    • docker rm $(docker ps -aq -f status=exited) && docker image prune -a
  • How can you list running containers using filters?
    • docker ps --filter status=running --format '{{.Names}} {{.Status}}'
  • Asses the status of docker daemon, docker disk usage, and docker containers?
    • docker system info docker system df
  • What is the default network driver for docker? How can you list docker networks?
    • # default network driver is bridge # overlay is the default network driver for swarm, it connects multiple Docker daemons together and enable swarm services to communicate with each other docker network ls
  • Check the logs of a container (in the last 60 minutes)?
    • docker logs --since 60m docker logs --since=1h
  • Get the logs of a specific container in a pod?
    • kubectl logs -f -c
